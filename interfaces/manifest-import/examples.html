TPEN Manifest Import Examples

Quick Start

The manifest import feature allows you to create projects from one or more IIIF manifests with a single link. Users are guided through authentication and see their newly created projects immediately.

Single Manifest Import

Import one IIIF manifest as a new project:

/import-manifest?manifest=http%3A%2F%2Fwww.e-codices.unifr.ch%2Fmetadata%2Fiiif%2Fsl-0002%2Fmanifest.json
Try It →

Multiple Manifest Import

Import multiple manifests as projects in one batch:

/import-manifest?manifest=URL1&manifest=URL2&manifest=URL3
Each manifest parameter should contain one IIIF manifest URL (URL-encoded).

How It Works

  1. User arrives with manifest URLs in the query string
  2. Authentication check - If not logged in, they're prompted to sign in
  3. Project creation - A loader shows while manifests are being imported
  4. Results display - Shows all created projects and any failures
  5. Quick actions - Links to view, transcribe, or manage projects

URL Encoding

IIIF manifest URLs must be URL-encoded. Use an encoder like urlencoder.org.

Original URL URL-Encoded
http://example.com/manifest.json http%3A%2F%2Fexample.com%2Fmanifest.json
https://example.org/path?query=value https%3A%2F%2Fexample.org%2Fpath%3Fquery%3Dvalue
Most modern browsers handle encoding automatically, but for maximum compatibility, always URL-encode manifest URLs.

Integration Examples

JavaScript

const manifestUrl = encodeURIComponent('http://example.com/manifest.json'); const importLink = `/import-manifest?manifest=${manifestUrl}`; window.location.href = importLink;

HTML Button

<a href="/import-manifest?manifest=http%3A%2F%2Fexample.com%2Fmanifest.json" class="btn"> Import Manifest </a>

Multiple Manifests in a Form

// Build URL with multiple manifests const manifests = [ 'http://example.com/manifest1.json', 'http://example.com/manifest2.json' ]; const params = new URLSearchParams(); manifests.forEach(m => params.append('manifest', m)); const url = `/import-manifest?${params.toString()}`;

Error Handling

The import process handles various error scenarios:

  • Not authenticated: User is prompted to sign in
  • Invalid manifest URL: Listed in failed section with error
  • Network error: Displayed with error details
  • Partial failures: Successfully created projects still show
  • No manifests in URL: Helpful message with examples

API Reference

The component uses the TPEN Services API to create projects from manifests:

POST /project/import?createFrom=URL Authorization: Bearer {token} Content-Type: application/json { "url": "https://example.com/manifest.json" }
Authentication is required. The component automatically handles token management.
